# Changelog

本项目采用按版本记录用户可见变化的方式维护 changelog。

## Unreleased

暂无。

## 1.3.5 - 2026-05-18

- 修复 Release 混淆包中软件更新清单解析被 R8 优化后始终显示“暂无发布版本”的问题。

## 1.3.4 - 2026-05-18

- 修复“一起看”请求复用影视站代理配置导致可能连接失败的问题。
- 改进“一起看”房间状态校验，避免缺失影片地址时加入房间崩溃。
- 修复 Release 混淆包中部分播放列表 JSON 解析可能报 `Abstract classes can't be instantiated` 的问题。
- 修复视频简介开头残留“简介”标签的问题。
- 改进手机端错误页和网站登录表单操作体验。

## 1.3.3 - 2026-05-18

- 修复软件更新检查可能因 GitHub REST API 未认证限流返回 HTTP 403 的问题。
- 发布流程新增 `update.json` 静态更新清单，客户端优先通过 Release 静态资产发现新版本，并保留 GitHub API 作为 fallback。
- 更新请求增加明确的 User-Agent，避免被 GitHub administrative rules 拒绝。

## 1.3.2 - 2026-05-18

- 修复软件更新检查在可安装 APK 已发布时仍可能显示“暂无发布版本”的问题。
- 更新检查现在会从最近的正式 Releases 中选择第一个带 APK 的版本，避免 latest 响应短暂无资产或非安装资产时误判。

## 1.3.1 - 2026-05-18

- 优化单集影片和单季剧集展示，避免显示多余的季标题和选集入口。
- 改进手机播放页遥控器向上键焦点行为，提升电视遥控操作体验。
- 优化“一起看”创建和加入房间界面。
- 将“视频源登录”文案统一为“网站登录”。
- 修复单季 wpse 播放列表解析时误附加季名的问题。

## 1.3.0 - 2026-05-17

- 新增“一起看”房间同步流程，并接入已部署的同步服务。
- 新增首页缓存加载，改善启动和列表加载体验。
- 升级数据库 schema 到版本 2，并补充缓存、迁移和同步模型测试。

## 1.2.8 - 2026-05-17

- 发布首个 GitHub 正式版本。
- 加入 GitHub Releases 应用内更新通道。
- 改进按季播放体验。
- 建立正式开源仓库文档、贡献指南、支持说明、安全策略、issue 模板和 CI。
